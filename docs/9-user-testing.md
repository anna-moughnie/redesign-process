# User Testing

This page covers the process of testing [our wireframes](./7-low-fi-prototypes.md) and the changes we planned to make based on our given feedback.

## Testing Process

We asked 3 people unfamiliar with our website to accomplish the goals outlined in [our user stories](./4-designing-for-users.md#user-stories). This would be done by allowing the tester to click through the site by handing them the pages they clicked on links for.

In this process, we refrain from guiding the user, and simply take notes when they make mistakes.

## Feedback

- Users skipped selecting a box size and went straight to selecting cupcakes.
  - We will address this by putting the step number in the headings for selecting box size and cupcakes (e.g. Step 1: Select Box Size).

- Users thought 'order' link in navigation menu was to place a new order.
  - We will rename this link 'my orders' to make its function immediately clear.

## Conclusion

With a clear plan to resolve issues brought up by testing, we were able to proceed with [building our high-fidelity prototypes](./10-hi-fi-prototypes.md).
